Cosine similarity over implicit rating vectors.
Σ sim(peer) × peer_weight(book).

Engineering Depth

Concurrency-Safe Borrow Logic

Two independent layers so concurrent requests can never corrupt available_copies

Layer 1 — Dialect-Aware Row Lock

PostgreSQL: SELECT … FOR UPDATE SKIP LOCKED — returns None immediately if locked → 409 Conflict (retryable).

SQLite: plain SELECT — WAL journal serialises concurrent writers at the DB engine level.

Layer 2 — Atomic Conditional UPDATE

The copy count is never read into Python and written back. All mutations use:

UPDATE book
  SET available_copies = available_copies - 1
  WHERE id = ? AND available_copies > 0

rowcount == 0 → another transaction raced and won → 409 Conflict.

Smart Hold Tracking (No Extra Column)

When a book is returned with a pending reservation, available_copies stays at 0 — the copy is implicitly held. Count of ready reservations = held copies. Zero extra DB columns needed.

Button States

● Copies available → Borrow
● No copies → Reserve
● In queue → Reserved #N
● Ready for me → Borrow (Ready)

If the ready user cancels: the held copy auto-passes to the next pending waiter — or increments available_copies if the queue is empty. Queue integrity is maintained without any background job.

Zero-Downtime Migrations

_migrate_db() runs on every startup. Uses add_missing_cols(table, additions) helper — calls ALTER TABLE only for columns not yet in the SQLite schema. New tables created by db.create_all().

Open Library + Pillow

Lazy scrape on first book view. Extracts description, author bio, cover image URL. Pillow downsamples cover to 64×64, bins mid-tone pixels, stores dominant #rrggbb in DB — used for instant theming without async canvas.

Engineering Judgement

Key Design Decisions

  • Session cookies over JWT — simpler for a monolith, Flask handles signing. Global Axios interceptor syncs React state on 401/403 to prevent stale "admin" UI after account switch.
  • Client-side filtering — all books loaded on mount, filtered in-memory with useMemo. Zero extra API calls on every filter change.
  • Cover colour extracted server-side (Pillow) — stored as #rrggbb in DB, served with book list. No async canvas extraction in the browser. WCAG-safe foreground computed at render.
  • Implicit borrow weight = 0.6 — intentionally below a 3-star rating (0.6 < 0.6̄) so passive reads don't dominate the preference profile.
  • Groq over OpenAI — sub-second inference for this catalogue size. temperature=0.1 for deterministic ranking. Hallucinated books structurally impossible (only returns DB IDs).
  • Collab dedup client-side — both rec endpoints stay independent and cacheable; React strips duplicates at render time.
  • Reactions as string keys, not emoji chars — avoids encoding issues; VALID_REACTIONS set is unambiguous; SVG icons map from keys.
